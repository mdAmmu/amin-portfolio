## Project Overview

M.A Bazaar is an innovative e-commerce solution designed specifically for local grocery stores. The project addresses the inefficiency of manual ordering processes in hyper-local markets by providing a digital storefront that supports role-based access for Owners, Employees, and Customers. 

Built with scalability and ease of use in mind, it bridges the gap between traditional retail and modern digital commerce through unique features like voice-to-order and bulk inventory management.

## Key Features

- **Voice-to-Order AI**: Customers can speak their order, which is then parsed into a structured cart using OpenAI/Speech Recognition.
- **Role-Based Dashboards**: Tailored interfaces for Admins (inventory/orders), Employees (delivery), and Users (shopping).
- **Bulk Product Upload**: High-performance Excel-to-Supabase migration for rapid inventory updates.
- **Billing & PDF Generation**: Automated bill generation with professional PDF exports using `jspdf`.
- **Hybrid Mobile Support**: Native Android integration via Capacitor for a seamless on-the-go experience.

## Technical Implementation

### Architecture

M.A Bazaar follows a modern BaaS (Backend-as-a-Service) architecture:
- **Frontend**: React (Vite) with Tailwind CSS for a responsive, mobile-first UI.
- **Backend/Database**: Supabase for real-time data sync, authentication, and Row-Level Security (RLS).
- **AI Integration**: Custom speech-to-text parsing and OpenAI service for natural language processing.
- **Mobile**: Capacitor for cross-platform app deployment.

## Challenges & Solutions

### Challenge 1: Voice Transcription Accuracy

**Problem**: The Web Speech API often provides incomplete transcripts, especially when users mention generic product names without quantities.

**Solution**: Implemented a fuzzy matching algorithm and a `numberMap` to normalize quantities (e.g., "two" to `2`). Used a multi-word scanning approach to match product names against the inventory with a scoring system.

### Challenge 2: Bulk Data Inconsistency

**Problem**: Importing inventory from Excel files often resulted in duplicate names or invalid number formats.

**Solution**: Developed a robust `BulkProductUpload` component that normalizes headers, filters empty rows, and performs batch upserts in Supabase with `onConflict: 'name'` logic to prevent duplicates.
