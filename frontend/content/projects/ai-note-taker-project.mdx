## Project Overview

AI Note Taker is a productivity-focused application designed to simplify
personal knowledge management. Many users store notes across different
platforms, making it difficult to organize information and retrieve
insights later. Traditional note-taking tools also lack intelligent
search and summarization capabilities.

This project addresses that problem by combining a fast modern web stack
with AI-powered text processing. Users can create notes, organize them
with tags, and automatically generate summaries and embeddings for
semantic search. The system is designed for students, developers, and
professionals who want a structured and searchable knowledge base.

The key innovation lies in combining vector search with a minimal,
responsive interface that allows users to quickly capture information
while still benefiting from advanced AI features.

## Key Features

-   **Smart Note Creation**: Users can create rich text notes with tags
    and categories for better organization.
-   **AI Summaries**: Automatically generates concise summaries of long
    notes using an LLM.
-   **Semantic Search**: Vector embeddings allow users to search notes
    by meaning rather than exact keywords.
-   **Tag-Based Organization**: Flexible tagging system for categorizing
    notes and filtering content.
-   **Fast Performance**: Optimized database queries and caching ensure
    quick retrieval even with large note collections.

## Technical Implementation

### Architecture

The system follows a modern full‑stack architecture:

-   Frontend: Next.js 14 with App Router
-   Backend: REST API using FastAPI
-   Database: PostgreSQL for structured note storage
-   Vector Storage: ChromaDB for semantic search
-   AI Services: OpenAI API for summarization and embeddings

The application separates responsibilities into clear layers:

1.  UI Layer -- handles user interactions
2.  API Layer -- processes note operations
3.  Data Layer -- manages relational and vector storage
4.  AI Layer -- generates embeddings and summaries

### Frontend

The frontend is implemented using Next.js and TypeScript to ensure type
safety and scalability. React components are structured using modular
design patterns to allow reusable UI components.

``` typescript
import { useState } from 'react';

type Note = {
  id: string;
  title: string;
  content: string;
};

const NotesList = () => {
  const [notes, setNotes] = useState<Note[]>([]);

  return (
    <div className="p-4">
      {notes.map(note => (
        <div key={note.id} className="border p-3 rounded mb-2">
          <h2 className="font-bold">{note.title}</h2>
          <p>{note.content.substring(0, 120)}...</p>
        </div>
      ))}
    </div>
  );
};

export default NotesList;
```

**Key Technologies:**

-   Framework: Next.js 14 with App Router
-   Styling: Tailwind CSS
-   State Management: React Query
-   UI Components: Custom component library

### Backend

The backend is implemented with FastAPI to provide high performance
asynchronous endpoints. The API handles note creation, updates, AI
processing, and search functionality.

``` python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Note(BaseModel):
    title: str
    content: str

notes = []

@app.post("/api/notes")
async def create_note(note: Note):
    notes.append(note)
    return {"message": "Note saved successfully"}

@app.get("/api/notes")
async def list_notes():
    return notes
```

**Infrastructure:**

-   API: FastAPI
-   Database: PostgreSQL
-   Authentication: JWT-based authentication
-   Hosting: Vercel (frontend) and Railway (backend)

### AI/ML Components

The AI layer enables semantic search and summarization features.

-   **Model Used**: GPT-4 for summarization
-   **Embeddings**: OpenAI text-embedding models
-   **Vector Database**: ChromaDB
-   **Prompt Engineering**: Structured prompts to summarize notes into
    concise bullet points

``` python
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()

vector_store = Chroma(
    collection_name="notes",
    embedding_function=embeddings
)
```

## Challenges & Solutions

### Challenge 1: Efficient Semantic Search

**Problem**: Searching thousands of notes using traditional SQL queries
is inefficient when users search conceptually rather than by keywords.

**Solution**: Implemented vector embeddings for each note and stored
them in ChromaDB. Semantic queries are converted into embeddings and
matched using cosine similarity, allowing meaningful search results.

### Challenge 2: Handling Long Notes

**Problem**: Long notes can exceed token limits for AI summarization.

**Solution**: Implemented a chunking strategy where long notes are split
into smaller segments before processing. Each chunk is summarized and
then combined into a final summary.
